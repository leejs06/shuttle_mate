# 셔틀메이트(ShuttleMate)

## 프로젝트 소개
      ㄴ 셔틀메이트는 배드민턴 경기 매칭 중심의 심플한 UI와 자동 매칭 경기 기능을 제공합니다.
### 주요 기능
      ㄴ 셔틀메이트는 모임 내에서 편리한 회원 관리와 경기 매칭 시스템을 중심으로 개발된 서비스 입니다.
### 기술 스택
      ㄴ Persistence: MyBatis(XML Mapper), MySQL DB
      ㄴ Tooling: DBeaver(DB Management), GitHub(VCS)
      ㄴ Deployment: Apache Tomcat
#### Backend
      ㄴ Java, Spring Boot
#### Frontend
      ㄴ JSP, JSTL(라이브러리), CSS, JavaScript
### 실행방법
      ㄴ Run 실행(shift + F10) - clean > package > 실행

### 일정 계획
   - 3월 3~4주: 개발환경 세팅(DB 생성 및 연동 포함), ERD 작성, 확실한 구조 이해
   - 4월 1~2주: JSTL 라이브러리를 활용하여, jsp로 UI 개발, css, js로 UI 디자인 및 동작 구현
   - 4월 3~4주: 톰캣 서버 아키텍처 분석 및 구성 (5/3~4주차와 변경됨)
   - 5월 1~2주: 테스트 및 학교에서 진행하는 중간평가 준비
   - 5월 3~4주: 밸런스 조정 로직 개발, MyBatis xml 쿼리 개발(4월 3~4주차와 변경됨)
   - 6월 1~2주: 운영 서버 배포 및 통합 테스트, 배포 자동화 점검 및 버그 수정
   - 6월 3~4주: 서비스 테스트, 기업/학교 최종평가 발표 및 시연
